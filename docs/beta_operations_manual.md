# β版運用マニュアル

やり投げ動画解析β版の運用手順書です。「申込受付」から「フィードバック集計・改善」までの14ステップをカバーします。

---

## 全体フロー

```
[1] 告知・SNS投稿
    ↓
[2] 申込フォーム受付
    ↓
[3] intake 登録
    ↓
[4] βテスター作成
    ↓
[5] 同意確認・連絡
    ↓
[6] 動画受け取り・確認
    ↓
[7] 解析ジョブ作成・実行
    ↓
[8] 解析確認・PDF・ダッシュボード生成
    ↓
[9] 納品（ダッシュボードURL送付）
    ↓
[10] フィードバック依頼
    ↓
[11] フィードバック確認・登録
    ↓
[12] 改善ログ作成
    ↓
[13] β版KPI確認
    ↓
[14] 正式版移行判断
```

---

## ステップ詳細

### [1] 告知・SNS投稿

1. `docs/social/beta_launch_posts.md` から対象ユーザーに合った投稿文を選ぶ
2. β版申込フォームのURLを貼り付ける
3. 「先着N名」の上限を明記する（`configs/beta_release_config.yaml` の `max_testers` に合わせる）
4. 投稿前に、β版であること・免責事項が含まれているか確認する

### [2] 申込フォーム受付

1. Google フォームの回答を定期的に確認する
2. 申込数が `max_testers` を超えた場合は、フォームを停止するか待機リストに追加する
3. 同意事項にチェックが入っているか確認する

### [3] intake 登録

1. 管理画面 → 「受付一覧」タブ → 「新規受付を作成」
2. 申込フォームの回答を入力する（`docs/beta_application_form_spec.md` のマッピング参照）
3. `source` は `"google_form"` を選択
4. `desired_plan` は `"beta_full_report"` を選択
5. ステータスを `needs_review` または `ready_for_job` に設定

### [4] βテスター作成

1. 管理画面 → 「βテスター」タブ → 「受付から作成」
2. intake_id を入力して「intakeからβテスターを作成」をクリック
3. 作成されたβテスターのステータスを `invited` に変更する
4. または「新規作成」から直接入力することも可能

### [5] 同意確認・連絡

1. 申込者の連絡先（LINE / DM）に参加確認メッセージを送る
2. 管理画面でβテスターの `consent_status` を `sent` → `agreed` に更新する
3. 未成年の場合は保護者・指導者への確認を依頼する
4. `tester_status` を `accepted` に変更する

**送付する内容（例）:**
```
β版にご参加いただけることになりました。
利用説明をご確認の上、動画をお送りください。
（docs/beta_user_explanation.md の内容を参照）
```

### [6] 動画受け取り・確認

1. 動画ファイルを受け取る（LINE / DM）
2. 動画が再生できることを確認する
3. 動画が `.mp4` 形式であることを確認する
4. 全身が映っていること、撮影角度を確認する

### [7] 解析ジョブ作成・実行

1. 管理画面 → 「新規ジョブ」タブ → 動画をアップロード
2. 身長が判明している場合は設定する
3. 解析モード: `all_variants`（推奨）
4. 「ジョブを作成して解析実行」をクリック
5. `job_id` を記録し、βテスターの `related_job_ids` に追加する
   - 管理画面 → βテスター → テスターを選択 → 「関連job_idを追加」
6. job.json に `is_beta=True` を設定する（update_job で kwargs として渡す）
7. βテスターのステータスを `active` に変更する

### [8] 解析確認・PDF・ダッシュボード生成

1. 解析が `completed` になっていることを確認する
2. 代表フレーム画像・グラフが生成されているか確認する
3. 管理画面 → ジョブ履歴 → 該当ジョブの「PDFレポート生成」をクリック
4. 管理画面 → ジョブ履歴 → 「Webダッシュボード生成」をクリック
5. ダッシュボードURLを確認する
6. ブラウザでダッシュボードが表示されることを確認する

### [9] 納品（ダッシュボードURL送付）

1. `generate_beta_delivery_message()` を使って納品メッセージを生成する
   ```python
   from src.message_templates import generate_beta_delivery_message
   msg = generate_beta_delivery_message(
       dashboard_url="https://example.com/dashboard/<token>",
       customer_label="＜名前／ニックネーム＞",
       feedback_form_url="<フィードバックフォームURL>",
   )
   ```
2. または管理画面 → ジョブ履歴 → 「β版納品メッセージ生成」（実装次第）
3. 申込者の連絡先（LINE / DM）にメッセージを送る
4. ジョブのステータスを `delivered` に変更する
5. βテスターのフィードバックステータスを `requested` に変更する

### [10] フィードバック依頼

1. 納品から3〜5日後にフォローアップメッセージを送る
2. フィードバックフォームへのリンクを再送する
3. 「フィードバックの提出期限（目安）」を伝える

### [11] フィードバック確認・登録

1. フィードバックフォームの回答を確認する
2. 管理画面 → 「フィードバック管理」タブ → 「フィードバックを作成」
   - または `src/feedback_manager.py` の `create_feedback()` で登録
3. βテスターの `feedback_status` を `submitted` → `reviewed` に更新する
4. βテスターの `tester_status` を `completed` に更新する

### [12] 改善ログ作成

1. 管理画面 → 「フィードバック管理」タブ → 対象フィードバックを選ぶ
2. 「📋 改善ログへ追加」ボタンをクリックする
3. または `src/improvement_log.py` の `create_improvement_from_feedback()` を使う
4. カテゴリ・優先度・担当フェーズを設定する
5. 重大バグ（critical）は最優先で対応する

### [13] β版KPI確認

1. 管理画面 → 「β版KPI」タブを確認する
2. 以下の指標を確認する:
   - テスター総数・完了数
   - フィードバック提出率
   - 緊急バグ件数
   - 改善ログの進捗

### [14] 正式版移行判断

以下の基準が全て達成されたら、正式版への移行を検討する。

- βテスター（同意済み）が 5 人以上
- フィードバック提出率が 70% 以上
- 緊急バグ（critical）が 0 件
- 納品済みジョブが 3 件以上
- 利用規約・プライバシーポリシーの法務確認が完了している

詳細は `docs/beta_release_checklist.md` を参照。

## 緊急対応フロー

### 重大バグが発生した場合

1. フィードバックの severity を `critical` に設定する
2. 管理画面の「β版KPI」タブで件数を確認する
3. 改善ログに `priority=critical` で登録する
4. 関係するβテスターに謝罪・状況説明メッセージを送る
5. 新規の申込受付を一時停止することを検討する

### 解析が失敗した場合

1. ジョブの stderr.txt でエラー内容を確認する
2. 動画に問題がある場合は `generate_video_issue_response()` メッセージを送る
3. システムエラーの場合はβテスターに謝罪し、再解析を試みる

---

> **ポリシーリマインダー**: フィードバック対応・SNS掲載確認・教師データ利用確認は、それぞれ独立した同意として管理してください。「解析に同意 = SNS掲載許可」ではありません。
